# A股恐惧贪婪指数（FGI）实施方案 · 终稿 V3.6

**版本**：V3.6（开发交付版）  
**更新日期**：2026年7月20日  
**设计原则**：数据免费、逻辑稳健、每日自动更新、可解释  

---

## 一、概述

### 1.1 指数定义

- **名称**：A股恐惧贪婪指数（A-Share Fear & Greed Index，下称 FGI）
- **分值含义**：0 = 极度恐惧（历史底部区域），100 = 极度贪婪（历史顶部区域）
- **更新频率**：交易日盘后自动计算，T 日数据最迟 T+1 日 08:00 前产出

### 1.2 设计目标

1. **全面捕捉情绪**：动量、情绪、估值、资金四大维度，13 个底层指标
2. **数据可落地**：全部指标均可免费获取，核心依赖 AKShare，爬虫类指标备有降级方案
3. **逻辑严谨**：处理共线性（季度监控），参数有分布依据（滚动分位数、MAD），杜绝硬编码
4. **工程可靠**：数据库增量更新，异常自动降级并报警，支持长期无人值守运行

### 1.3 版本演进

V3.6 基于 V3.5 的最终评审意见，修复以下问题：

- **P1**：F3 年化基差率到期日爆炸问题修复（去掉年化因子，增加到期切换机制）
- **P2**：择时策略增加连续 3 日确认与 20 日冷却期
- **P2**：S3 ±1 周同比描述改为伪代码明确算法
- **P3**：资金维度指标权重标签统一为 8.33%/8.33%/8.34%，与公式一致
- **P3**：S1/S4 平滑尺度差异增加回测验证说明

> **权重说明**：四维度等权（各 25%），维度内指标等权。权重直接加总为 100%，无需额外归一化。一致性调节因子作为独立后处理步骤，不占用权重。

### 2.2 各指标详细定义

#### M1 融资买入占比（Z‑score + sigmoid）

- **原始值**：`融资买入额 / (上证成交额 + 深市全市场成交额)`
- **处理**：  
  1. 计算滚动 5 年（1200 交易日）均值与标准差，得 Z‑score。  
  2. 得分映射：`score = 100 / (1 + e^{-Z})`（sigmoid，全区间单调连续）。
- **方向**：融资越活跃 → 得分越高（贪婪）。

#### M2 全A上涨家数占比

- **原始值**：`上涨家数 / (上涨家数 + 下跌家数)`
- **处理**：直接对原始比值做滚动 5 年百分位。
- **方向**：上涨占比越高 → 得分越高。

#### M3 上证偏离 60 日均线

- **原始值**：`(上证收盘价 – 60日均线) / 60日均线`
- **处理**：滚动 5 年百分位。
- **方向**：正偏离越大 → 得分越高。
- **标的选择说明**：选用上证指数而非沪深300，因上证指数在散户认知中代表“大盘”，情绪锚定效应更强，更符合行为金融学逻辑。

#### M4 创 60 日新高/新低家数归一化差

- **原始值**：`(创60日新高家数 – 创60日新低家数) / (创60日新高家数 + 创60日新低家数)`
  - 若分母为 0，取值为 0。
- **处理**：滚动 5 年百分位。
- **方向**：正值越大 → 强势股主导 → 得分越高（贪婪）。

#### S1 50ETF 平值 IV 均值

- **计算步骤**：  
  1. 获取 50ETF 当日收盘价及当月、下月所有期权合约。  
  2. 筛选平值附近看涨、看跌各两档。  
  3. 以 Shibor 或 10 年期国债收益率为无风险利率，BS 反推 IV。  
  4. 取算术平均。
- **降级**：若计算失败 → 自动切 50ETF 20 日历史波动率。
- **得分**：IV 做滚动 5 年百分位，**反向**（IV 越高 → 得分越低）。
- **注**：S1 与 S4 平滑尺度不同（日值 vs 20日均值），上线后回测需验证一致性，必要时统一加 5 日 EMA。

#### S2 散户入场意愿（双频融合）

- **子指标A（周频）**：  
  - 数据源：巨量算数（主）+ 百度指数（备），“股票开户”周均搜索量。  
  - 处理：两源各自独立计算 **滚动 2 年百分位** → 得分。  
  - 合成：两源均成功取得分均值；仅一源成功用该源。  
  - 每日填充：最近一周得分。
- **子指标B（月频）**：  
  - 来源：中国结算新增自然人投资者，6 月移动平均环比增速。  
  - 处理：滚动 5 年百分位得分。  
  - 每日填充：最近月得分。
- **融合**：A、B 得分各占 50%，得 S2 最终得分。
- **降级**：A 连续 2 周失败 → 仅用 B，S2 权重全部分配给 B。

#### S3 东财上证指数吧热度（交易周同比）

- **爬取目标**：上证指数吧，统计每日发帖量。
- **去趋势方法**：  
  1. 按交易日周序号聚合周发帖总量（不足 3 个交易日的周与相邻周合并）。  
  2. 计算比值，分母算法如下（伪代码）：
     
     ```
     yearly_avgs = []
     for year in (今年-5) .. (今年-1):
         weeks_in_year = [第N-1周, 第N周, 第N+1周] 在该年的发帖量
         year_avg = mean(weeks_in_year)
         yearly_avgs.append(year_avg)
     denominator = mean(yearly_avgs)
     ratio = current_week_posts / denominator
     ```
  3. 对该比值做滚动 5 年百分位。
- **方向**：比值越高 → 得分越高（贪婪）。

#### S4 50ETF成交量PCR（20日均值）

- **原始值**：`认沽成交量 / 认购成交量`，20 日简单移动平均。
- **得分**：滚动 5 年百分位，**反向**（PCR 越低 → 得分越高）。
- **注**：S4 与 S1 平滑尺度不同，回测阶段需验证是否统一为日值。

#### V1 沪深300风险溢价（ERP）

- **原始值**：`1 / 沪深300滚动市盈率 – 10年期国债收益率`
- **得分**：滚动 5 年百分位，**反向**（ERP 高 = 估值低 = 恐惧 → 得分低）。

#### V2 ΔERP Z‑score（250 日窗口，方向为负向）

- **原始值**：`ΔERP = ERP_t − ERP_{t-1}`
- **处理**：  
  1. 取过去 250 个交易日的 ΔERP 序列，计算 Z‑score。  
  2. 映射：`score = 100 / (1 + e^{Z})`（指数取 +Z 实现负向）。
- **方向**：**负向**（ΔERP 为负 → 得分高，代表贪婪）。

#### F1 ETF净申赎额（20日累计）

- **原始值**：`Σ[(当日份额 – 前日份额) × 单位净值]`，20 日累计。
- **得分**：滚动 5 年百分位。
- **方向**：**默认正向**（+）。**强制验证**：回测阶段计算其与未来收益的 Rank IC，若显著为负则翻转为反向。
- **方向确认后处理**：若 F1 确认为反向指标，资金维度内部将重构为两个子维度：  
  - 机构行为（F1 + F2，均为反向，权重各半）  
  - 衍生品情绪（F3，正向）  
    两者等权聚合，避免信号抵消。

#### F2 大宗交易折价率（20日加权均值）

- **原始值**：过去 20 个交易日，单笔 ≥ **500 万元**的大宗交易的“成交价相对收盘价折价率”，成交额加权平均。
- **得分**：滚动 5 年百分位，**反向**（折价率越高 → 得分越低）。

#### F3 沪深300期指原始基差率

- **原始值**：
  - 计算 `(近月合约价格 / 沪深300指数 − 1)` 作为原始基差率。
  - **到期处理**：若近月合约距离到期日 ≤ 5 个交易日，则自动切换至次月合约。
- **得分**：滚动 5 年百分位，**正向**（贴水收窄/升水扩大 = 乐观 → 得分高）。
- **设计说明**：不进行年化，避免临近到期时年化因子爆炸导致百分位剧烈跳变。

## 三、合成方法

### 3.1 得分标准化

- 各指标按 2.2 节方法转化为 0–100 得分。
- 缺失值用最近有效得分填充，最长不超过 5 个交易日。

### 3.2 第一层：维度内聚合

- 动量维度得分 = `(M1 + M2 + M3 + M4) / 4`
- 情绪维度得分 = `(S1 + S2 + S3 + S4) / 4`
- 估值维度得分 = `(V1 + V2) / 2`
- 资金维度得分 = `(F1 + F2 + F3) / 3`（若 F1 方向确认并启用子维度结构，则按 2.2 节 F1 说明重构）

### 3.3 第二层：加权合成原始 FGI

```
FGI_raw = 0.25 × 动量 + 0.25 × 情绪 + 0.25 × 估值 + 0.25 × 资金
```

（权重合计 100%，无需额外归一化）

### 3.4 第三层：条件一致性调节（后处理）

- 计算每日 **13 个指标得分**的 **MAD**（中位绝对偏差）。
- 计算该 MAD 在滚动 5 年中的**分位数** `p`（0~1）。
- **仅当 FGI_raw < 15 或 FGI_raw > 85 时**，启动调节：
  
  ```
  λ = min( p × 0.6 , 0.3 )
  FGI_final = FGI_raw × (1 − λ) + 50 × λ
  ```
- 其他情况 `FGI_final = FGI_raw`。

**参数说明**：

- `0.6` 为强度系数，`0.3` 为最大回归幅度。
- 须在样本内（2015–2022）验证调节因子的正向贡献，若无效则移除。

## 四、数据获取与工程实现

### 4.1 技术栈

- **语言**：Python 3.9+
- **依赖**：`akshare`, `pandas`, `numpy`, `requests`, `beautifulsoup4`, `scipy`, `sqlite3`, `matplotlib`
- **存储**：SQLite
- **调度**：Linux cron 或 Windows 任务计划，交易日 18:00 触发

### 4.2 数据库设计（EAV 模式）

```sql
-- 原始数据统一存储
CREATE TABLE raw_data (
    date TEXT,
    indicator TEXT,        -- 如 'margin_ratio', 'iv', 'pcr', 'futures_basis' …
    value REAL,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (date, indicator)
);

-- 每日最终得分
CREATE TABLE scores_daily (
    date TEXT PRIMARY KEY,
    M1 REAL, M2 REAL, M3 REAL, M4 REAL,
    S1 REAL, S2 REAL, S3 REAL, S4 REAL,
    V1 REAL, V2 REAL,
    F1 REAL, F2 REAL, F3 REAL,
    FGI_raw REAL, FGI_final REAL
);

-- 每日数据状态
CREATE TABLE daily_status (
    date TEXT,
    indicator TEXT,
    status TEXT,  -- 'normal', 'missing', 'degraded', 'substituted'
    PRIMARY KEY (date, indicator)
);
```

### 4.3 每日运行流程

1. 确定最近交易日 T。
2. 逐指标增量更新，写入 `raw_data`，同时更新 `daily_status` 表。
3. 从 `raw_data` 读取足够历史，计算得分及 FGI，写入 `scores_daily`。
4. 容错降级：见各指标降级方案；连续缺失 5 日则剔除，权重分配给同维度其他指标。
5. 异常日志 + 报警（企业微信/钉钉 Webhook）。
6. **计算并输出指数健康度**（见 4.5 节），健康度低于 60 时在推送中标注“数据质量异常，仅供参考”。
7. **FGI 日度变动异常检测**：计算当日 FGI_final 与前一交易日的绝对差值。若该值超过滚动 5 年日度变动的 99% 分位数，自动标记为“疑似异常”，暂停自动发布，触发人工复核。

**版本兼容与回滚**：每次大版本升级（如 V4.0），数据库保留新旧两套得分的计算逻辑，并行运行至少 3 个月。`scores_daily` 表增加 `FGI_legacy` 与 `FGI_current` 两个字段，通过配置开关切换主版本，旧版逻辑保留 6 个月以备回滚。

### 4.4 冷启动方案

- **回填起点**：2015‑01‑01。
- 前 5 年使用扩展窗口；2020 年起固定 5 年滚动窗口。
- 各指标数据起始时间及处理见附录 B。

### 4.5 指数健康度评分

每日同时输出 0–100 的健康度，基于：

- **正常指标占比 × 50**
- **(1 – 降级占比) × 30**
- **(1 – 相关系数超标率) × 20**  
  健康度低于 60 时，FGI 值旁标注“数据质量异常，仅供参考”。

## 五、回测与验证（开发阶段必须完成）

### 5.1 样本分割

- **样本内**：2015–2022（用于参数选择和指标优化）
- **样本外**：2023–2026（纯验证，不作任何调整）  
  样本外若 IC 不显著或分层单调性差，指数不可上线。
- **额外滚动验证**：以 2 年为训练窗、1 年为测试窗，从 2018 年开始滚动，报告平均表现，增强结论稳健性。

### 5.2 分层回测

- 对每一交易日 T，仅用 T 之前数据确定 FGI 的 10 档等分阈值。
- 统计每档未来 5/20/60 日的收益均值和胜率。
- 基准：沪深300、中证500、中证1000 全收益指数。
- 检验单调性，前 3 年（2015–2017）标注参考性有限。

### 5.3 IC 分析

- 每半年用过去 3 年数据计算 FGI 及各指标与未来 20 日收益的 Rank IC。
- 多重检验校正：13 指标 × 3 窗口 = 39 次检验，采用 Bonferroni 校正。
- 输出 IC 均值、标准差、IR（IC 均值/标准差）、胜率。

### 5.4 策略模拟

- **逆情绪定投**：每月定投金额 = 基础金额 × (1 − FGI/100) × 2。
- **极端阈值择时**：
  - 信号触发条件：FGI 需**连续 3 个交易日** < 15（买入）或 > 85（卖出）。
  - 冷却期：触发买入后，至少持有 **20 个交易日**才允许反向信号生效。
  - 计入双边 0.3% 交易成本。
- 对比基准：
  - 买入持有
  - 20 日均线择时
  - 60 日均线择时
  - 股债性价比模型（基于 ERP 历史分位阈值）
- 统计年化收益、最大回撤、年换手次数、信号频率（年均买入/卖出次数）。

### 5.5 其他验证

- **F1 方向验证**：计算 ETF 净申赎与未来收益的 IC，决定最终方向；若反向则按 2.2 节 F1 说明重构资金维度。
- **一致性调节有效性**：对比有/无调节的样本外表现。
- **剔除测试**：逐一去掉单个指标，观察 FGI 整体 IC 变化。IC 降幅低于 FGI 整体 IC 的 10%，视为贡献微小，进入淘汰观察。
- **S1/S4 平滑验证**：回测阶段比较 S4 日值与 20 日均值的表现，决定是否统一平滑尺度。

## 六、上线后的监控与迭代

- **季度相关性检查**：维度内相关 > 0.75、跨维度 > 0.65 触发人工审查。
- **波动率窗口评估**：每季度检查 S1、S4 的分布漂移，必要时改用 3 年窗口或衰减加权。
- **数据源健康度**：每周报告缺失次数与降级状态。
- **权重定期审视**：上线后每年度可基于过去一年各维度的 Rank IC 表现，在 ±5% 范围内微调维度权重，避免完全僵化。
- **未来扩展**：  
  - 权重 IC 动态化、多市场版本（V4.0 计划引入 VIX 或恒生指数风险溢价，权重 5%）、引入雪球/基金申赎等另类数据。

## 七、局限性声明

1. **市场结构变化**：5 年窗口可能滞后于注册制、量化扩容等体制变化。波动率类指标（S1、S4）建议持续监控。
2. **共线性处理**：依赖季度人工监控，未引入自动正交化。
3. **另类数据风险**：爬虫指标可能因网站改版中断。
4. **历史数据限制**：部分指标早期缺失（见附录 B），冷启动期精度略降。

## 附录 A：关键公式速查

- **Z‑score**：`Z = (x – μ) / σ`，μ, σ 为滚动窗口均值与标准差
- **Sigmoid 映射（正向）**：`score = 100 / (1 + e^{-Z})`
- **Sigmoid 映射（负向）**：`score = 100 / (1 + e^{Z})`
- **滚动百分位**：`rank_pct = (rank – 1) / (N – 1) × 100`
- **归一化差（M4）**：`(新高家数 − 新低家数) / (新高家数 + 新低家数)`
- **F3 原始基差**：`(近月合约 / 沪深300 − 1)`，到期 ≤5 日切换次月
- **一致性因子**：  
  `MAD = median(|Xi − median(X)|)`，`p = rolling_percentile(MAD, 5yrs)`  
  仅当 FGI_raw ∈ [0,15) ∪ (85,100]：`λ = min(p×0.6, 0.3)`，`FGI_final = FGI_raw×(1−λ) + 50×λ`
- **健康度**：`正常占比×50 + (1−降级占比)×30 + (1−超标率)×20`
