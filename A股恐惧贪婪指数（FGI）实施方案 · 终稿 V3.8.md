# A股恐惧贪婪指数（FGI）实施方案 · 终稿 V3.8

**版本**：V3.8.1（逻辑修正版）  
**更新日期**：2026年7月22日  
**设计原则**：数据免费、逻辑稳健、每日自动更新、可解释  

> **V3.8.1 修订**（2026-07-22，基于数据层审计与用户决策）：
> - 删除 S1 涨跌比：其实现与 M2 严格线性（S1 = 2×M2 − 1），数据库实证得分逐日相同，构成重复计权。情绪维度只留 S2、S3，各占 12.5%。指标总数 12 → 11。
> - 维度整体缺失时跨维度权重重归一化，不再使用默认 50 分。

## 一、概述

### 1.1 指数定义

- **名称**：A股恐惧贪婪指数（A-Share Fear & Greed Index，下称 FGI）
- **分值含义**：0 = 极度恐惧（历史底部区域），100 = 极度贪婪（历史顶部区域）
- **更新频率**：交易日盘后自动计算，T 日数据最迟 T+1 日 08:00 前产出

### 1.2 设计目标

1. **全面捕捉情绪**：动量、情绪、估值、资金四大维度，11 个底层指标
2. **数据可落地**：全部指标均可免费获取，核心依赖 AKShare + levistock + zzshare，爬虫类指标备有降级方案
3. **逻辑严谨**：消除指标同源重复，保留利率锚，确保维度内逻辑自洽
4. **工程可靠**：数据库增量更新，异常自动降级并报警，支持长期无人值守运行

### 1.3 版本演进

V3.8 基于 V3.7 深度评审，修正以下核心问题：

- **P1‑1**：删除与 M2/S1 同源的 S2（散户情绪），情绪维度权重重新分配，消除信号重复放大
- **P1‑2**：V1 恢复为 ERP（1/PE – 10年国债收益率），保留利率锚，更准确衡量股债性价比
- **P1‑3**：V2 恢复为 ΔERP Z‑score（250日窗口），与 V1 构成估值水平+变化速度的双指标互补
- **P1‑4**：F3 概念修正为“主力资金板块偏好”，明确定义与历史代理策略，标注不确定性
- **P2**：M1/S4 预期高相关预先标注并预设处理方案；F2 周频滞后、F1 代理偏差补充说明

### 2.2 各指标详细定义

#### M1 涨停板家数

- **原始值**：当日涨停板股票家数（含一字板、炸板后封板）
- **数据源**：levistock `stock_zt_pool_em`（主），AKShare `stock_zt_pool_em`（备，仅历史）
- **处理**：直接对原始值做滚动 5 年百分位。
- **方向**：涨停家数越多 → 得分越高（贪婪）。
- **降级**：levistock 失败 → AKShare（仅限历史日期 < 今日）。
- **注**：与 S3（涨停封单量）源自同一数据源，上线后若两者相关系数 > 0.85，则将 S3 权重减半，差额并入 S2。

#### M2 散户意愿（上涨家数占比）

- **原始值**：`上涨家数 / (上涨家数 + 下跌家数)`
- **数据源**：zzshare `open_sentiment`（主），AKShare 全市场涨跌家数（备）
- **处理**：滚动 5 年百分位。
- **方向**：上涨占比越高 → 得分越高。
- **说明**：本指标衡量市场宽度。（V3.8.1 起 S1 涨跌比已删除：其实现 `(up−down)/(up+down)` 与本指标 `up/(up+down)` 严格线性，得分逐日相同，构成重复计权。）

#### M3 上证偏离 60 日均线

- **原始值**：`(上证收盘价 – 60日均线) / 60日均线`
- **数据源**：AKShare `stock_zh_index_daily` (sh000001)
- **处理**：滚动 5 年百分位。
- **方向**：正偏离越大 → 得分越高。
- **标的选择说明**：选用上证指数而非沪深300，因上证指数在散户认知中代表“大盘”，情绪锚定效应更强。

#### M4 创业板换手率偏离

- **原始值**：创业板指数 (sz399006) 换手率的 Z‑score（相对 60 日均值与标准差）
- **数据源**：AKShare `stock_zh_index_daily` (sz399006)
- **处理**：Z‑score → 滚动 5 年百分位。
- **方向**：换手率显著高于均值 → 投机情绪高涨 → 得分越高。

#### S2 股吧热度

- **原始值**：zzshare `market_hot_sentiment` 返回的 `p_close` 字段（市场热度指数收盘值）
- **数据源**：zzshare `market_hot_sentiment`
- **处理**：滚动 5 年百分位。
- **方向**：热度越高 → 得分越高（贪婪）。

#### S3 涨停封单量

- **原始值**：当日所有涨停板股票的封单金额总和（亿元）
- **数据源**：levistock `stock_zt_pool_em`（主），AKShare `stock_zt_pool_em`（备，仅历史）
- **处理**：滚动 5 年百分位。
- **方向**：封单量越大 → 得分越高。
- **降级**：同 M1。
- **注**：与 M1 同源，已预先制定相关性处理方案（见 M1 注）。

#### V1 沪深300风险溢价（ERP）

- **原始值**：`1 / 沪深300滚动市盈率 – 10年期国债收益率`
- **数据源**：AKShare `stock_index_pe_lg` (沪深300) + `bond_china_yield`
- **处理**：滚动 5 年百分位，**反向**（ERP 高 = 股票相对债券便宜 = 恐惧，得分低）。
- **角色**：估值安全垫与股债性价比锚。

#### V2 ΔERP Z‑score（250 日窗口，方向为负向）

- **原始值**：`ΔERP = ERP_t − ERP_{t-1}`
- **处理**：  
  1. 取过去 250 个交易日的 ΔERP 序列，计算 Z‑score。  
  2. 映射：`score = 100 / (1 + e^{Z})`（指数取 +Z 实现负向）。
- **方向**：**负向**（ERP 快速下行 → 股票急涨 → 贪婪，得分高）。
- **说明**：V2 衡量估值变化速度，与 V1 估值水平形成互补。

#### F1 融资余额占比

- **原始值**：`上交所融资余额 / 上证总市值`（简化代理；优先尝试两市合计值，若深交所数据获取困难则用此代理）
- **数据源**：AKShare `stock_margin_sse`
- **处理**：滚动 5 年百分位。
- **方向**：融资越活跃 → 得分越高。
- **注**：代理偏差已标注于局限性声明。

#### F2 基金股票仓位

- **原始值**：公募基金股票仓位中位数（百分比）
- **数据源**：AKShare `fund_stock_position_lg`（周频，每周五更新）
- **处理**：  
  1. 周频数据前向填充至日频。  
  2. 对日频序列做滚动 5 年百分位。
- **方向**：仓位越高 → 机构越乐观 → 得分越高。
- **局限**：周频填充在极端行情下信号滞后可达 5 个交易日，已标注于局限性声明。

#### F3 主力资金板块偏好

- **原始值**：全市场行业资金流净额汇总（亿元）。`stock_fund_flow_industry` 返回 90 个行业的净额，全市场净流入 = Σ(净额)。
- **数据源**：AKShare `stock_fund_flow_industry`（当日真实数据，主）；上证指数量价代理（历史期，备）。
- **处理**：  
  1. **当日值**：真实行业资金流汇总。  
  2. **历史值**：2026‑07‑21 之前无行业资金流历史，沿用上证指数 `|price_change × volume|` 代理序列。  
  3. **平滑过渡**：两段序列在 2026‑07‑21 拼接，滚动百分位自动融合。前 2 年（2024‑07‑21 前）的百分位参考性有限，在回测中标注。
- **方向**：净流入越大 → 得分越高。
- **概念说明**：本指标衡量主力资金在板块层面的配置方向，而非个股大单净流入，两者可能存在差异。

## 三、合成方法

### 3.1 得分标准化

- 各指标按 2.2 节方法转化为 0–100 得分。
- 缺失值用最近有效得分填充，最长不超过 5 个交易日。

### 3.2 第一层：维度内聚合

- 动量维度得分 = `(M1 + M2 + M3 + M4) / 4`
- 情绪维度得分 = `(S2 + S3) / 2`
- 估值维度得分 = `(V1 + V2) / 2`
- 资金维度得分 = `(F1 + F2 + F3) / 3`

### 3.3 第二层：加权合成原始 FGI

```
FGI_raw = 0.25 × 动量 + 0.25 × 情绪 + 0.25 × 估值 + 0.25 × 资金
```

（权重合计 100%，无需额外归一化）

**维度缺失处理（V3.8.1）**：某维度全部指标缺失时，该维度不计入（**不使用默认 50 分**），其余维度权重按比例放大；全部维度缺失则当日不产出 FGI。

### 3.4 第三层：条件一致性调节（后处理）

- 计算每日 **11 个指标得分**的 **MAD**（中位绝对偏差）。
- 计算该 MAD 在滚动 5 年中的**分位数** `p`（0~1）。
- **仅当 FGI_raw < 15 或 FGI_raw > 85 时**，启动调节：
  
  ```
  λ = min( p × 0.6 , 0.3 )
  FGI_final = FGI_raw × (1 − λ) + 50 × λ
  ```
- 其他情况 `FGI_final = FGI_raw`。

**参数说明**：

- `0.6` 为强度系数，`0.3` 为最大回归幅度。
- 须在样本内（2015–2022）验证调节因子的正向贡献，若无效则移除。

## 四、数据获取与工程实现

### 4.1 技术栈

- **语言**：Python 3.9+
- **依赖**：`akshare`, `levistock`, `zzshare`, `pandas`, `numpy`, `requests`, `beautifulsoup4`, `scipy`, `sqlite3`, `matplotlib`
- **存储**：SQLite
- **调度**：Linux cron 或 Windows 任务计划，交易日 18:00 触发

### 4.2 数据库设计（EAV 模式）

```sql
CREATE TABLE raw_data (
    date TEXT,
    indicator TEXT,        -- 如 'zt_count', 'fund_position', 'industry_fund_flow' …
    value REAL,
    update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (date, indicator)
);

CREATE TABLE scores_daily (
    date TEXT PRIMARY KEY,
    M1 REAL, M2 REAL, M3 REAL, M4 REAL,
    S1 REAL, S2 REAL, S3 REAL,  -- S1 已废弃（V3.8.1 删除），列保留兼容历史数据
    V1 REAL, V2 REAL,
    F1 REAL, F2 REAL, F3 REAL,
    FGI_raw REAL, FGI_final REAL
);

CREATE TABLE daily_status (
    date TEXT,
    indicator TEXT,
    status TEXT,  -- 'normal', 'missing', 'degraded', 'substituted'
    PRIMARY KEY (date, indicator)
);
```

### 4.3 每日运行流程

1. 确定最近交易日 T。
2. 逐指标增量更新，写入 `raw_data`，同时更新 `daily_status` 表。
3. 从 `raw_data` 读取足够历史，计算得分及 FGI，写入 `scores_daily`。
4. 容错降级：见各指标降级方案；连续缺失 5 日则剔除，权重分配给同维度其他指标。
5. 异常日志 + 报警（企业微信/钉钉 Webhook）。
6. **计算并输出指数健康度**（见 4.5 节），健康度低于 60 时推送标注“数据质量异常，仅供参考”。
7. **FGI 日度变动异常检测**：当日 FGI_final 与前一交易日的绝对差值若超过滚动 5 年日度变动的 99% 分位数，标记“疑似异常”，暂停自动发布，触发人工复核。

**版本兼容与回滚**：每次大版本升级，数据库保留新旧两套得分，并行运行至少 3 个月。`scores_daily` 表增加 `FGI_legacy` 与 `FGI_current` 字段，通过配置开关切换主版本，旧版逻辑保留 6 个月以备回滚。

### 4.4 冷启动方案

- **回填起点**：2015‑01‑01。
- 前 5 年使用扩展窗口；2020 年起固定 5 年滚动窗口。
- 各指标数据起始时间及处理见附录 B。

### 4.5 指数健康度评分

每日输出 0–100 的健康度，基于：

- **正常指标占比 × 50**
- **(1 – 降级占比) × 30**
- **(1 – 相关系数超标率) × 20**  
  健康度低于 60 时，FGI 值旁标注“数据质量异常，仅供参考”。

## 五、回测与验证（开发阶段必须完成）

### 5.1 样本分割

- **样本内**：2015–2022（用于参数选择和指标优化）
- **样本外**：2023–2026（纯验证，不作任何调整）  
  样本外若 IC 不显著或分层单调性差，指数不可上线。
- **额外滚动验证**：以 2 年为训练窗、1 年为测试窗，从 2018 年开始滚动，报告平均表现。

### 5.2 分层回测

- 对每一交易日 T，仅用 T 之前数据确定 FGI 的 10 档等分阈值。
- 统计每档未来 5/20/60 日的收益均值和胜率。
- 基准：沪深300、中证500、中证1000 全收益指数。
- 检验单调性，前 3 年（2015–2017）标注参考性有限。

### 5.3 IC 分析

- 每半年用过去 3 年数据计算 FGI 及各指标与未来 20 日收益的 Rank IC。
- 多重检验校正：12 指标 × 3 窗口 = 36 次检验，采用 Bonferroni 校正。
- 输出 IC 均值、标准差、IR（IC 均值/标准差）、胜率。

### 5.4 策略模拟

- **逆情绪定投**：每月定投金额 = 基础金额 × (1 − FGI/100) × 2。
- **极端阈值择时**：
  - 信号触发：FGI 需**连续 3 个交易日** < 15（买入）或 > 85（卖出）。
  - 冷却期：触发买入后，至少持有 **20 个交易日**才允许反向信号。
  - 计入双边 0.3% 交易成本。
- 对比基准：
  - 买入持有
  - 20 日均线择时
  - 60 日均线择时
  - 股债性价比模型（基于 ERP 历史分位阈值）
- 统计年化收益、最大回撤、年换手次数、信号频率。

### 5.5 其他验证

- **F1 方向验证**：计算融资余额占比与未来收益的 IC，默认正向。
- **一致性调节有效性**：对比有/无调节的样本外表现。
- **剔除测试**：逐一去掉单个指标，观察 FGI 整体 IC 变化，IC 降幅 < FGI 整体 IC 的 10% 则进入淘汰观察。
- **M1/S3 相关性处置验证**：若上线后两者相关系数 > 0.85，按预设方案降权，并对比降权前后 FGI 的 IC。
- **F3 混合策略影响评估**：前 2 年（2024‑07‑21 前）的百分位受代理值影响，在回测中单独统计该期间的 IC。

## 六、上线后的监控与迭代

- **季度相关性检查**：维度内相关 > 0.75、跨维度 > 0.65 触发人工审查。
- **波动率窗口评估**：每季度检查 V2 等波动率类指标的分布漂移，必要时改用 3 年窗口或衰减加权。
- **数据源健康度**：每周报告缺失次数与降级状态。
- **权重定期审视**：每年度可基于过去一年各维度的 Rank IC 表现，在 ±5% 范围内微调维度权重。
- **未来扩展**：  
  - 权重 IC 动态化、多市场版本（V4.0 计划引入 VIX 或恒生指数风险溢价）、引入 ETF 申赎、雪球结构化产品情绪等。

## 七、局限性声明

1. **市场结构变化**：5 年窗口可能滞后于注册制、量化扩容等体制变化。波动率类指标建议持续监控。
2. **共线性处理**：依赖季度人工监控，未引入自动正交化。M1 与 S3 同源已预设应对方案。
3. **另类数据风险**：zzshare、levistock 等爬虫数据源可能因网站改版中断。
4. **F1 代理偏差**：融资余额占比简化为上交所数据，未涵盖深交所，存在小幅偏差。
5. **F2 周频滞后**：基金仓位为周频前向填充，极端行情下信号滞后可达 5 个交易日。
6. **F3 混合策略**：历史期使用上证指数代理，早期百分位参考性有限（前 2 年已标注）。
7. **历史数据限制**：部分指标早期缺失（见附录 B），冷启动期精度略降。

## 附录 A：关键公式速查

- **Z‑score**：`Z = (x – μ) / σ`，μ, σ 为滚动窗口均值与标准差
- **Sigmoid 映射（正向）**：`score = 100 / (1 + e^{-Z})`
- **Sigmoid 映射（负向）**：`score = 100 / (1 + e^{Z})`
- **滚动百分位**：`rank_pct = (rank – 1) / (N – 1) × 100`
- **ERP**：`1/沪深300PE – 10年国债收益率`
- **一致性因子**：  
  `MAD = median(|Xi − median(X)|)`，`p = rolling_percentile(MAD, 5yrs)`  
  仅当 FGI_raw ∈ [0,15) ∪ (85,100]：`λ = min(p×0.6, 0.3)`，`FGI_final = FGI_raw×(1−λ) + 50×λ`
- **健康度**：`正常占比×50 + (1−降级占比)×30 + (1−超标率)×20`
